# Swarms Cloud 全新升级发布

过去几天里，我们为 Swarms Cloud 发布了一大批重要升级。本次发布的核心，是全新的 Token 用量仪表盘——它能让团队实时掌握 Token 消耗与成本情况，同时我们还重新设计了模型目录（Model Catalog），如今已覆盖来自行业领先厂商的 1,500 多个模型。

这些改动共同指向一个目标：让个人开发者和企业团队都能更轻松地监控用量、为特定任务找到合适的模型、放心地部署智能体（Agent），并在一个统一平台上大规模运行多智能体工作流。本文将详细介绍本次发布的具体内容、其重要性,以及如何立即开始使用这些新功能。

## Token 用量可观测性

随着多智能体系统从原型阶段迈向生产环境，团队最常问的问题已经不再是"这套系统能不能跑通"，而是"这到底花了我们多少钱，钱花在了哪里"。一旦并行运行的智能体和工作流数量超过几个，如果没有细粒度的 Token 消耗可见性，这个问题就很难回答。

全新的 Token Usage 仪表盘正是为解决这个问题而生。它为工程团队和财务团队提供了一个共享的、权威的视图，可以精确掌握 Token、请求量与支出在组织内各个智能体、工作流以及 API 流量之间的分布情况。

本次发布的核心能力包括：

- **用量追踪。** 在单一视图中监控整个组织的 Token 消耗、请求量与整体 AI 支出，而不必再从日志或临时脚本中手动拼凑数据。
- **智能体级别的分析。** 按单个智能体和工作流细分用量数据，让团队可以清晰识别出系统中哪些部分正在拉高成本，以及优化投入将在哪里产生最大效果。
- **灵活的筛选功能。** 按时间范围、API Key、智能体或工作流对用量数据进行切片分析，从而既能回答宽泛的问题（"上个月我们总共花了多少钱"），也能回答具体的问题（"是哪个 API Key 导致了这次用量激增"）。
- **实时成本可见性。** 实时追踪基础设施成本与支出模式，而不必等到月末账单出来才搞清楚预算都花在了哪里。

对于运行在受监管或成本敏感环境中的团队而言，这种级别的可观测性往往是将多智能体工作负载迁移到生产环境的必要前提，而不只是锦上添花的功能。这个仪表盘的设计目标，正是让这一迁移过程无需依赖任何定制化工具即可完成。

立即前往 [cloud.swarms.world/token-usage](https://cloud.swarms.world/token-usage) 体验。

## 全新模型页面

为某项任务挑选合适的模型，再正确地完成接入，过去往往需要在厂商文档、定价页面与反复试错的集成代码之间来回切换。全新设计的 Swarms Cloud 模型页面（Model Pages），正是为了把整个流程收束成一套单一且一致的体验。

模型目录如今已覆盖来自 Anthropic、OpenAI、Google Gemini、OpenRouter 等主流厂商的 1,500 多个模型，全部可以在同一个地方进行搜索与比较。

全新体验包括：

- **模型发现。** 按厂商、类别与能力浏览完整目录，快速定位适合特定应用场景的模型——无论是长上下文推理、低延迟对话，还是多模态输入。
- **即用型代码示例。** 每个模型页面都配有可直接运行的 Python、TypeScript 与 cURL 示例代码，接入工作从一份可运行的代码开始，而不是一张空白文件。
- **单智能体与多智能体支持。** 目录中的任何模型，既可以作为独立智能体部署，也可以组合进更大规模的多智能体团队中，无需切换工具或重写配置。
- **Playground 集成。** 在编写任何接入代码之前，直接在浏览器中测试模型的实际表现，这在比较多个候选模型完成同一任务时，能大幅缩短评估周期。

前往 [cloud.swarms.world/models](https://cloud.swarms.world/models) 浏览完整模型目录。

## 其他改进

除了这两项重点功能之外，本次发布还包含了一系列虽小但同样有意义的平台改进：

- **Auto Generate Prompts 支持指定模型。** 团队现在可以为自动生成提示词功能指定具体使用的模型，而不再局限于固定的默认模型，从而对提示词质量和成本拥有更多控制权。
- **主页仪表盘优化。** 仪表盘现在可以一目了然地展示真实的任务完成数量与当前可用额度总量，让用户一登录就能清楚了解账户状态。
- **订阅管理。** 现在可以直接在设置页面升级套餐，无需再通过客服支持来完成这类常规账户变更。
- **UI 与 UX 改进。** 根据用户持续反馈，我们在整个平台上推出了数十项 Bug 修复、性能优化与易用性改进。

这些改动单独来看都只是渐进式的，但整体上大幅减少了用户在平台上日常运行和管理智能体时所遇到的种种阻力。

## 为生产级工作流而生

贯穿本次所有更新的一条主线，是对生产就绪（Production Readiness）的高度重视。Token 用量可观测性为团队提供了在规模化场景下负责任运营所需的成本与性能信号。模型目录缩短了从"决定尝试一个新模型"到"真正将其上线"之间的时间。订阅管理与仪表盘改进则降低了单纯运行一个账户所带来的运营负担。

这一切都并非偶然。随着企业组织从单智能体实验迈向处理真实业务负载的协同多智能体系统，围绕这些系统的工具链也必须与智能体本身同步成熟。本次发布，正是对这一趋势的直接回应。

## 结语

这些更新是我们迈向"打造最强大的生产级多智能体 AI 系统构建、部署与管理平台"这一目标的又一步。从模型发现到用量分析，再到日常账户管理，本次发布中的每一项功能，都旨在为开发者和企业带来更强的掌控力、更高的可见性，以及更大的灵活性。

我们将持续投入 Swarms Cloud 的建设，不断推出新能力，让大规模部署和运营 Swarms 变得越来越简单。
