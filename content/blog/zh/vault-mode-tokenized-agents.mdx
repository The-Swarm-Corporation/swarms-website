# Vault 模式：如何把代币化智能体变成一个持币门槛型产品

[Swarms Marketplace](https://swarms.world) 上的代币化智能体或提示词,本质上是一个普通 listing,只是多了一个属性:它在 Solana 上拥有自己可交易的代币,创作者可以从这枚代币的每一笔交易中持续抽成。如果你还没读过代币化、债券曲线和创作者手续费如何运作的完整介绍,可以先看这篇文章:[代币化智能体：是什么、如何发布，以及创作者如何获得持续收益](https://swarms.world/blog/tokenized-agents-launch-revenue)。本文假设你已经了解这些基础知识,专注深入讲解这个流程中的一项具体功能:Vault 模式（Vault Mode）。

## Vault 模式到底是什么

默认情况下,一个代币化 listing 的代币纯粹是金融属性的。任何人都可以免费使用底层的智能体或提示词(或者按标价付费使用),与此同时,任何人也都可以在债券曲线上买卖它的代币。这两件事本来互不相关。

Vault 模式改变了这一点。它是发布时可选的一个开关,就在[发布页面](https://swarms.world/launch)代币化设置区里 Frenzy 模式（Frenzy Mode）的旁边,开启后代币本身就变成了访问机制。开关旁边的应用内说明写得很直白:

> "只有持币者才能使用这个{entityType}。用户必须购买代币才能访问它。"

开启后,底层的智能体或提示词就会被锁在一道持币检查之后。根据产品自身的解锁页面文案,门槛是在已连接的钱包中持有价值至少 1 美元的该代币。买入代币、持有它,内容就会解锁;卖出代币,下一次检查时内容会重新锁定。

## 门槛机制如何端到端运作

Vault 模式由一个名为 `VaultGate` 的客户端组件(`shared/components/entity/parts/vault-gate.tsx`)负责执行,它包裹的是智能体或提示词页面的整个内容区域,而不只是其中一部分。这个门槛只在两个条件同时满足时才会生效:该 listing 的 `vault_mode` 字段为开启状态,并且该 listing 已经代币化。只要有一个条件不满足,或者代币地址尚不存在,门槛就不会生效,页面会正常渲染。

关于这项检查具体如何运作,有几个值得注意的细节:

- **创作者始终能看到自己的内容。** 门槛机制会显式跳过 listing 所有者,所以发布一个带 Vault 模式的智能体,永远不会把你自己挡在门外。
- **检查针对的是你连接的钱包。** 当一位非所有者的访客打开一个带 Vault 模式的页面时,组件会调用一个服务端接口 `/api/vault-mode/check-holder`,传入访客的钱包地址和代币地址。这个接口会查询该钱包的链上代币余额,并返回该钱包是否持有这枚代币。
- **结果会缓存五分钟。** 检查结果会在客户端(内存中以及会话存储中)按「钱包 + 代币」为键缓存五分钟,这样就不用每次渲染都重新检查。锁定页面里有一个手动刷新按钮,可以立即强制重新检查,这在刚完成购买之后很关键,否则一个缓存的旧结果可能会再持续几分钟。
- **未连接钱包等同于没有访问权限。** 如果访客压根没有连接钱包,会被当作非持有者处理。这里并没有单独的「请连接钱包」引导页;锁定视图本身就承担了连接钱包的引导作用,因为里面内置的购买组件本来就需要连接钱包才能使用。

## 锁定和解锁状态到底长什么样

被锁定的内容并不是彻底隐藏起来的。真实的页面内容其实就在下面,只是被模糊化、去饱和、禁用交互,上面覆盖着一张「Vault Mode: Holders only(仅限持币者)」的卡片。这张卡片会显示该 listing 的评分、访客当前的代币余额(带一个刷新按钮)、代币地址(带复制按钮和 Solscan 链接),以及一个内置的购买组件:一个 SOL 金额输入框、几个快捷购买金额按钮、一个实时报价,以及一个「购买」按钮,点击后会直接从访客已连接的钱包发起兑换,无需离开当前页面。旁边还有一个链接,可以跳转到 Axiom 上交易这枚代币,供更习惯在那里交易的人使用。

一旦持币检查通过,无论是因为访客本来就持有代币,还是刚通过购买组件买到了代币,界面都会先播放一段简短的「Access granted(访问已开通)」提示动画,然后平滑地移除模糊效果,原地展示完整的智能体或提示词内容,全程无需刷新页面。

## Vault 模式与定价方式互斥

Vault 模式取代的是美元定价,而不是叠加在美元定价之上。如果你在一个已设为「付费」的 listing 上开启 Vault 模式,发布页面会自动把定价切回「免费」,并弹出提示解释原因:Vault 模式已经通过持币量控制了访问权限,在此之上再收取一份美元价格是多余的。反过来也是同样强制的:只要 Vault 模式处于开启状态,「付费」这个定价选项就会被直接禁用,如果尝试发布一个既是付费又开启了 Vault 模式的 listing,系统会在提交时直接拦截报错。

Frenzy 模式则是完全独立的另一回事。它是一个让创作者的手续费分成翻倍的开关,发布流程中没有任何机制阻止它与 Vault 模式同时启用。创作者完全可以同时发布一个既开启 Vault 模式(使用需要持币)、又开启 Frenzy 模式(交易量分成翻倍)的 listing。

## 如何发布一个带 Vault 模式的智能体或提示词

1. 前往[发布页面](https://swarms.world/launch),像平时一样填写你的智能体或提示词:名称、描述和内容。
2. 在「定价」部分打开代币化开关。这项功能只对智能体和提示词开放,工具不支持。
3. 设置 Ticker Name（代码符号）,并选择 Quote Mint（计价币种,SOL 或 USDC)。
4. 在同一个代币化面板里,把 Vault 模式的开关打开。如果你的 listing 之前设为「付费」,定价会自动切回「免费」。
5. 如果你还想同时获得双倍手续费分成,可以选择性地也打开 Frenzy 模式。
6. 如果还没有连接 Solana 钱包,先连接它;代币化要求发布前必须连接钱包。
7. 点击发布。按提示支付一次性的代币发布费用,然后尽快买入自己的代币,因为创建代币并不会自动把任何代币分配给你。

## 创作者为什么选择 Vault 模式

普通付费 listing 和带 Vault 模式的 listing 都会为访问收费,但收费背后的机制完全不同,而这种差异正是考虑使用 Vault 模式的全部理由:

- **它把使用需求直接转化为代币买入压力。** 在普通付费 listing 里,买家付一次钱,交易就结束了。而在带 Vault 模式的 listing 里,每一个想使用这个智能体或提示词的人,都必须先去买入代币,这份需求会直接体现在债券曲线上,而不是体现为一次性的账单。
- **它让代币的价值与真实使用价值对齐。** 一枚只依附于纯粹投机性 listing 的代币,只有叙事价值。而一枚为了解锁人们真正想用的东西而必须持有的代币,其需求下限和底层产品本身绑定在一起,而不只是取决于交易情绪。
- **它给了持币者继续持有的理由。** 在一个普通的代币化 listing 里,没有什么能阻止买家一有小幅盈利就立刻卖出。而开启 Vault 模式后,卖出代币就意味着失去使用这个工具的权限,这就给那些实际在使用该智能体或提示词的人,提供了一个持续持有、而不是频繁买卖的理由。

## 买家如何在市场上识别带 Vault 模式的 Listing

Vault 模式在任何人尝试打开锁定页面之前就是可见的。无论是在市场的网格视图,还是在具体的 listing 详情页,只要 Vault 模式处于开启状态,标题旁边就会显示一个带锁形图标的「VAULTED」小徽章。市场注册表页面的「定价」筛选器里也有一个专门的「Vault」选项,和「免费」「付费」「已代币化」「Frenzy」并列,买家可以据此把整个注册表筛选到只显示带 Vault 模式的 listing,或者反过来把它们排除掉。

## 常见问题

**用过一次智能体之后,是不是还得一直持有代币?**
是的。持币检查会在你之后每次访问时重新运行(受五分钟缓存影响),所以能否持续访问,取决于你是否持续持有不低于门槛要求的代币数量。

**如果我压根没有连接钱包会怎样?**
你会被当作非持有者,看到锁定视图,其中内置的购买组件会在你尝试使用它时提示你连接钱包。

**Vault 模式可以和 Frenzy 模式一起使用吗?**
可以。它们是两个相互独立的开关。Vault 模式通过持币量控制访问权限,Frenzy 模式则改变创作者在交易量上的手续费分成。没有任何限制阻止两者同时开启。

**Vault 模式支持工具(Tool)吗?**
不支持。代币化本身只对智能体和提示词开放,而 Vault 模式脱离代币化没有意义,因此也继承了同样的限制。

## 开始使用

Vault 模式是整个代币化流程中的一个可选功能,特别适合你希望让代币真正为使用付费把关,而不只是纯粹用于投机交易的场景。

- **发布一个带 Vault 模式的智能体或提示词:** 前往[发布页面](https://swarms.world/launch),打开代币化开关,并一并打开 Vault 模式。
- **看看现在已经上线的带 Vault 模式的 listing:** 在[市场注册表](https://swarms.world/platform/registry)中按 Vault 筛选。
- **阅读完整的代币化指南:** 关于债券曲线、手续费分成和 Frenzy 模式的详细说明,请参阅[代币化智能体：是什么、如何发布，以及创作者如何获得持续收益](https://swarms.world/blog/tokenized-agents-launch-revenue)。
